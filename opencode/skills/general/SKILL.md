# General Research Guidelines

## When to Use
- Unclear location of info/code; need to fan out searches
- Complex questions needing multiple passes or sources
- Multi-step tasks with dependencies between findings

## Approach
- Start with scope: what is known vs unknown; note assumptions
- Prioritize sources: codebase > tests > docs > issues/PRs > external
- Iterate: search, summarize, identify gaps, refine query
- Log paths searched and key hits; avoid re-searching blindly

## Output Style
- Concise summary first, then supporting details
- Cite file paths and line numbers for code; link or quote for docs
- Call out uncertainties and next queries if gaps remain

## Common Mistakes
- Shallow single search; fix by iterating with refined terms
- No citations; fix by listing paths/lines or sources
- Mixing speculation with facts; fix by labeling assumptions explicitly
